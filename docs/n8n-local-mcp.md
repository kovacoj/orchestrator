# n8n Local MCP Integration (Signal Foundry)

This workspace treats n8n as a local MCP-exposed workflow runner.

Architecture split:

- OpenCode orchestrator calls n8n MCP tools.
- n8n executes refresh/schedule workflows and delivery hooks.
- Signal Foundry backend owns intelligence and session state.

## Allowed MCP tools

Expose only these tools:

- `sf_refresh_session`
- `sf_get_session_alerts`
- `sf_trigger_demo_monitor`

Do not expose arbitrary workflow or credential capabilities.

## Local prerequisites

- Signal Foundry backend expected at `http://localhost:8000`
- n8n expected at `http://localhost:5678`

Start n8n:

```bash
docker compose up -d n8n
```

## MCP wiring in OpenCode

Set these env vars locally:

- `N8N_MCP_URL` = exact endpoint generated by n8n MCP Server Trigger
- `N8N_MCP_TOKEN` = token configured in n8n MCP auth

The repo config already includes a `n8n-local` MCP server that reads these values.

## Workflow contract

Create these workflows in n8n UI and expose them as MCP tools:

1. `Signal Foundry - Refresh Session`
2. `Signal Foundry - Get Session Alerts`
3. `Signal Foundry - Trigger Demo Monitor`
4. `Signal Foundry - Scheduled Monitor` (not MCP-exposed)

Tool behavior expectations:

- `sf_refresh_session`
  - input: `session_id` (required), `scenario` (optional), `mode` (optional)
  - calls `POST /sessions/{session_id}/refresh`
  - returns normalized `{ ok, tool, session_id, prediction_changed, new_headline, alert }`
- `sf_get_session_alerts`
  - input: `session_id` (required)
  - calls `GET /sessions/{session_id}/alerts`
  - returns normalized `{ ok, tool, session_id, alerts }`
- `sf_trigger_demo_monitor`
  - input: `scenario` (required), `session_id` (optional)
  - minimum fallback when no session bootstrap endpoint exists:
    - return `{ ok: false, error.type: "MissingSessionId" }`

## Current backend gap

This orchestrator repository does not currently include a FastAPI backend exposing:

- `POST /sessions`
- `POST /sessions/{session_id}/run`
- `POST /sessions/{session_id}/refresh`
- `GET /sessions/{session_id}/cards`
- `GET /sessions/{session_id}/alerts`

Until those endpoints exist in the backend service, n8n workflows can only be validated structurally or against stubs.

## Local validation checklist

1. Backend process reachable at `http://localhost:8000`.
2. `docker compose up -d n8n` starts successfully.
3. n8n MCP endpoint copied exactly into `N8N_MCP_URL`.
4. OpenCode can list `n8n-local` MCP with only approved tools.
5. Manual n8n run of `Signal Foundry - Refresh Session` returns normalized payload.
6. Scheduled workflow triggers refresh and gates on `prediction_changed === true`.
